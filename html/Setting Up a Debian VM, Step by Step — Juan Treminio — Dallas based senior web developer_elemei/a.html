                                            Eztrenk • 3 years ago

                                            I tried your guide, but I'm not getting past the point of setting up the second Network Adapter. Any attempt to SSH in to the VM is getting a timeout.

                                            The windows hosts file contains the correct command, the interfaces file has only the lines you specified added and I've added exceptions to the Windows Firewall for Virtualbox and KiTTy, both inbound and outbound to always be allowed. I've even turned the firewall off, restarted and tried it again, but still I get a timeout.

                                            Running "sudo ifconfig -a" shows eth1 is running and has the correct IP address.
                                            I can still run a regular "sudo apt-get update" or ping www.google.com. But the second interface is seemingly not working properly.

                                            Please advise proper action.

                                            EDIT: problem solved, let's hope it's permanent.

                                            What happened is I didn't understand how Virtualbox configures the Host-Only network Adapter.

                                            See, I thought this was just some technology that Virtualbox employed that would always work for all VM's installed.

                                            Instead, it turns out you can actually make any Host-Only network configuration(under File -> Preferences -> Network). This is done via the Virtualbox manager, not via the individual VM GUI. The problem was that the default Host-Only network for me was not set to 192.168.56.1, it had some exotic IP address I've already forgotten, something alone the lines of 169.224.x.x

                                            Changing it didn't seem to work though, the changes were never saved, they always returned to the exotic address.

                                            So, I had to make a new one with the correct settings and tell the Host-Only Adapter for the Debian VM to use those settings.

                                            The correct settings being show here: http://i.imgur.com/f4MyVgL.png
                                            And here: http://i.imgur.com/su9iA64.png

                                            I took those images from this page: https://forums.virtualbox.org/...

                                            That's how I fixed it: remove the default host-only network and make a new one with the correct settings.

                                            I'm not guaranteeing this will work for anyone else, I'm just saying it worked for me.
